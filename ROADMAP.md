# Roadmap

Super Skills is a public collection of Markdown-based skills for AI coding agents. The goal is to keep the skills practical, portable, and useful without locking them to one agent app or workflow.

## Current Status

- `v0.1.0` public launch is live.
- 21 standalone SUPER Skills are published as individual repositories.
- Each skill can be installed and used independently.
- The master catalogue links the full series and keeps discovery simple.

## Near Term

- Improve install guidance for different local agent setups.
- Add more real-world example prompts to the most-used skills.
- Add issue templates for bug reports, skill requests, and improvement ideas.
- Add lightweight checks for skill metadata, broken links, and README consistency.
- Tighten the master catalogue so new users can pick the right skill faster.

## Skill Improvements

- Strengthen routing guidance inside each skill so agents choose the right module faster.
- Add more domain-specific examples and expected outputs.
- Improve validation checklists for audits, reviews, and implementation work.
- Refine module boundaries when a skill becomes too broad or repetitive.
- Keep main `SKILL.md` files concise and move deeper workflows into referenced modules.

## Refresh Cadence

The collection is intended to be reviewed every 2-3 months.

Priority areas for recurring updates:

- AI coding workflows and agent patterns
- AI SEO, GEO, LLM visibility, and search platform changes
- Cloud, security, and DevOps best practices
- Platform SDK changes for commerce, CRM, bots, and automation tools
- Design, accessibility, and frontend implementation patterns

## Possible Future Additions

- More example packs for each skill.
- Optional install scripts for common agent environments.
- Public compatibility notes for popular AI coding setups.
- Community-contributed modules after review.
- A small validation script for checking skill structure before release.

## Non-Goals

- Do not replace official platform documentation.
- Do not make the skills dependent on one AI coding app.
- Do not move large reference material into the main `SKILL.md`.
- Do not split skills just because a module is large.
- Do not add generated filler, vague advice, or untested workflows.
