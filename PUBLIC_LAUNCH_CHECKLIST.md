# Public Launch Checklist

Use this checklist before publishing or promoting a SUPER Skills release.

## Repository Basics

- README explains what the skill does, who it is for, and when to use it.
- `SKILL.md` is the installable entry point.
- `references/modules/` files are bundled when the skill depends on deeper workflows.
- `VERSION` and `CHANGELOG.md` match the latest release.
- Licence is present and visible.

## Discovery

- GitHub description is clear, keyword-friendly, and specific.
- Repository topics include `super-skills`, `agent-skills`, `ai-coding`, `vibe-coding`, and relevant domain topics.
- README includes practical search terms naturally, without keyword stuffing.
- The catalogue links to the standalone skill repository.
- The standalone skill links back to the catalogue.

## Installation And Usage

- `INSTALL.md` explains generic installation for Markdown-compatible AI agents.
- README includes a short install example.
- Example prompts show realistic tasks.
- Compatibility language stays platform-neutral.
- Related SUPER Skills are positioned as optional enhancements, not hard dependencies.

## Product Quality

- Modules are useful without requiring hidden or external private files.
- Guardrails are clear for sensitive domains such as healthcare, legal, finance, and security.
- Outputs are concrete: checklists, audits, implementation plans, templates, or remediation steps.
- Language is public-facing, not internal project notes.
- British English is used consistently where practical.

## GitHub Readiness

- Issue templates exist for bugs, improvements, new skill requests, and documentation feedback.
- Roadmap is published and realistic.
- Social preview image or README hero image is present where useful.
- Release tag exists for the current version.
- Release notes explain what is included and how to install.

## Feedback Loop

- README or issue templates make it clear where feedback should go.
- Improvement requests ask for the task, expected output, and missing workflow.
- New skill requests ask for the domain, use cases, and example prompts.
- Roadmap leaves room for regular upgrades every few months.
